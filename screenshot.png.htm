Fetch and Checkout

Wait a minute! You just realized that one of the requirements for the README was to include a screenshot of the blog app! You forgot to add that in, but you’ve already deleted your local branch.

You need to get the add-readme branch off of GitHub exactly as is and into your local repo so you can keep working on it.

The first thing you should do here is close your own pull request so that this detail isn’t overlooked and merged in accidentally.

Go to your pull request in GitHub and close it for now:

Now you need to get your branch which now only lives in the remote GitHub repo into your local Git repo.

Run the following commands:

git fetch
git checkout -b add-readme origin/add-readme

This should first fetch the remote branches, caching them for later use. Then it should create and checkout to a new local branch named add-readme, but rather than copying the current branch to create it, the remote add-readme branch will be copied instead.

You can verify this worked by making sure the README.md file is present in your local project folder.

If you wanted to test out and run a teammates code on your machine instead of just looking at it in the browser during a code review, you’d follow this method of fetching and then checking out the origin branch to get it on your machine.

To complete this issue, either take a screenshot of your complete blog app or use the one below:

Save it to the root of your repo as "screenshot.png"

Then add the following line to the end of the README.md file:

<img src="screenshot.png" alt="Blog Screenshot">

And save.

Add and commit your changes:

git add -A
git commit -m "Add screenshot to README"

Then push this branch back up to GitHub

git push origin add-readme

From GitHub, resubmit the pull request. You should be able to select the add-readme branch from the dropdown and click the "Pull request button"

Merge the pull request into master.

To recap this lesson, git fetch and git checkout can be used to bring down a copy of a remote branch that doesn’t exist locally. This is valuable for updating a pull request after you’ve deleted the local branch for it, or for retrieving a teammates branch for review on your machine.
